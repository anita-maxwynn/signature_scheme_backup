\doxysubsection{Detailed Description}
Implementation of parameter handling functions for the signature scheme. 

This file provides functions for initializing, generating, reading, and handling parameters used in the signature scheme. It manages parameters for generator matrices and concatenated codes, supports random and user-\/defined parameter generation, and handles persistent storage of parameters.

Main functionalities include\+:
\begin{DoxyItemize}
\item Initialization of the libsodium library for cryptographic randomness.
\item Generation of random parameters within specified ranges.
\item Interactive user input for code parameters, including support for BCH code parameters.
\item Reading and writing parameter sets to a persistent file.
\item Accessor functions for retrieving current parameter values.
\end{DoxyItemize}

The parameters managed include those for two generator matrices (G1, G2) and a concatenated code (H\+\_\+A).

The function will repeatedly ask the user for input until valid parameters are provided (i.\+e., n \texorpdfstring{$>$}{>} k and n \texorpdfstring{$>$}{>} d). The validity of the parameters is checked to ensure that {\ttfamily n} is either greater than {\ttfamily k} or {\ttfamily d}. If the input is invalid, it will prompt the user to try again.

void get\+\_\+user\+\_\+input (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{structParams}{Params}} \texorpdfstring{$\ast$}{*}}]{g1}{, }\item[{\mbox{\hyperlink{structParams}{Params}} \texorpdfstring{$\ast$}{*}}]{g2}{, }\item[{\mbox{\hyperlink{structParams}{Params}} \texorpdfstring{$\ast$}{*}}]{h}{}\end{DoxyParamCaption})}



Get user input for parameters. 

This function checks if a saved parameter file exists and prompts the user to use it. If not, it asks the user whether they want to use BCH code or input G1 and G2 parameters manually. It generates random parameters if the user chooses not to input them. The function also saves the parameters to a file for future use. At first it checks if a saved parameter file exists. If it does, it prompts the user to use it. If the user chooses not to use the saved parameters, it asks whether they want to use BCH code or input G1 and G2 parameters manually.
\begin{DoxyItemize}
\item For the BCH code, it calculates the parameters based on user input for {\ttfamily m} and {\ttfamily t}, ensuring that the derived values for {\ttfamily n}, {\ttfamily k}, and {\ttfamily d} are consistent across G1 and G2. The parameters for H\+\_\+A are derived from G1 and G2. {\ttfamily m} is the degree of the BCH code, and {\ttfamily t} is the error-\/correcting capability. The parameters are calculated as follows\+:
\item -\/ n = 2\texorpdfstring{$^\wedge$}{\string^}m -\/ 1 (length of the codeword)
\item -\/ k = m \texorpdfstring{$\ast$}{*} t (length of the message)
\item -\/ d = 2 \texorpdfstring{$\ast$}{*} t + 1 (minimum distance of the code)
\item If the user chooses to input G1 and G2 parameters manually, it prompts for each parameter (n, k, d) and checks their validity. If the user does not want to input parameters, it generates random parameters for G1 and G2.
\end{DoxyItemize}

After gathering the parameters, it saves them to a file named Defined as PARAM\+\_\+\+PATH for future use. The parameters for G1, G2, and H\+\_\+A are printed to the console for confirmation.
